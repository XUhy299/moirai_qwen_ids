本文件按时间倒序记录对 `ids/moirai_qwen_ids/` 下代码的所有修改。每次修改必须包含：日期、修改人、涉及文件、变更摘要、变更原因。

## 2026-07-21 — 修复 discrete-to-text 批次维度 bug（严重）

**修改人**: Claude (研究执行助手)

**涉及文件**:
- `mqids/model.py` — `MoiraiQwenClassifier.forward`
- `mqids/prompting.py` — `PromptAssembler.forward` 和 `_encode_actuator`

**变更摘要**:

1. `mqids/model.py:92-112`:
   - `actuator_text: str | None` → `actuator_texts: list[str] | None`
   - 对批次中每个样本独立生成 actuator_text: `for i in range(batch_size)` 循环调用 `build_discrete_text`
   - 不再只取 `windows_disc[0]`

2. `mqids/prompting.py:59-102`:
   - `PromptAssembler.forward` 新增 `actuator_texts: list[str] | None` 参数
   - 新增 `_encode_actuator_batched` 方法: 逐样本编码、右填充到最大长度、生成 padding mask
   - `forward` 中构建分段 attention_mask: prefix(全1) + actuator(mask) + soft_tokens(全1) + suffix(全1)

**变更原因**:

原有代码 `windows_disc[0].T` 只取批次第一个样本生成文本，然后 `_encode_actuator` 将同一段文本广播到整个 batch。训练时 batch_size=4 → 75% 样本拿到错误的执行器文本；评估时 batch_size=16 → 94% 样本拿到错误的执行器文本。模型无法从离散文本中学习有效信号。

修复后每个样本独立生成自己的离散变量描述，变长文本通过右填充 + attention mask 处理，确保每个样本只看到自己的离散状态信息。

## 2026-07-21 — 简化离散变量模板

**修改人**: Claude (研究执行助手)

**涉及文件**:
- `mqids/switch_templates.py`

**变更摘要**:
- 二值变量模板: 从复杂的切换描述（"在第N个时间步由关闭切换为开启，之后保持开启"）简化为仅输出当前状态 `{变量名}：关闭` / `{变量名}：开启`
- 低基数变量模板: 从多状态变化描述简化为 `{变量名}：目前值为状态N`
- 移除 `_generate_binary_summary`、`_generate_lc_summary` 两个内部函数，逻辑合并到 `generate_discrete_summary`
- 保留 `_unique_states`、`_state_name`、`build_discrete_text` 作为公共接口

**变更原因**:
用户要求简化模板，离散变量只需描述当前状态值，不需要窗口内的切换历史。

## 2026-07-21 — 离散变量自然语言化（discrete-to-text）

**修改人**: Claude (研究执行助手)

**涉及文件**:
- `mqids/data.py` — `ChannelMetadata` 的 `continuous_indices`、`discrete_indices`、`discrete_names`、`discrete_type_ids` 属性
- `mqids/switch_templates.py` — **新文件**，离散变量状态序列的中文自然语言模板生成
- `mqids/model.py` — `MoiraiQwenClassifier.__init__` 和 `forward`，支持分离连续/离散变量
- `mqids/prompting.py` — `PromptAssembler.forward` 支持可选的 actuator_text 中插
- `mqids/config.py` — `ExperimentConfig` 新增 `discrete_to_text: bool = False`
- `mqids/factory.py` — `build_model` 新增 discrete-to-text 参数，MOIRAI target_dim 按连续变量数计算
- `scripts/train.py` — 新增 `--discrete-to-text` CLI 参数，protocol 记录 discrete_to_text

**变更摘要**:

1. `mqids/switch_templates.py`（新文件）:
   - `generate_discrete_summary(name, values, type_id)` → 单变量单行中文句子
   - 二值变量（type=1）: "关闭/开启" 状态 + 是否存在切换 + 切换时间步
   - 低基数变量（type=2）: "状态N" + 变化次数 + 最后变化时间步
   - `build_discrete_text(names, windows, type_ids)` → 多样本天然语言块

2. `mqids/prompting.py`:
   - `PromptAssembler.__init__` 保存 `self._tokenizer` 引用
   - `PromptAssembler.forward` 新增 `actuator_text: str | None = None` 参数
   - actuator_text 存在时插入顺序: prefix → actuator_text → soft_tokens → suffix

3. `mqids/model.py`:
   - `MoiraiQwenClassifier.__init__` 新增 `discrete_to_text`、`discrete_names`、`_discrete_type_ids`、`_continuous_indices`、`_discrete_indices`
   - `forward`: 当 `discrete_to_text=True` 时，窗口按列拆分为连续部分（走 MOIRAI）和离散部分（生成文本），离散文本通过 `actuator_text` 传入 prompt

4. `mqids/data.py`:
   - `continuous_indices` / `discrete_indices`: 修复为返回 active 子集内的位置索引（0..84），而非原始 110 维索引
   - `discrete_names`: 通过 `active_indices` 反查原始变量名

5. `mqids/config.py`:
   - `ExperimentConfig` 新增 `discrete_to_text: bool = False`

6. `mqids/factory.py`:
   - `build_model` 新增 `continuous_indices`、`discrete_indices`、`discrete_names`、`discrete_type_ids` 参数
   - `discrete_to_text=True` 时 MOIRAI `target_dim` = `len(continuous_indices)`（仅连续变量）
   - 无 Qwen 基线不支持 discrete-to-text

7. `scripts/train.py`:
   - 新增 `--discrete-to-text` 可选 CLI flag
   - `build_model` 调用传入 metadata 的离散变量属性
   - protocol 记录 `discrete_to_text`

**变更原因**:

根据研究方案设计，离散变量（开关量、低基数状态量）不应作为连续值输入 MOIRAI，而应转化为自然语言描述放在 Prompt 前缀中，让 Qwen 的语言理解能力处理这些结构化状态信息。该功能通过可选 CLI flag 控制，默认关闭以保持向后兼容。

## 2026-07-21 — 通道恒定判定扩展至全量数据

**修改人**: Claude (研究执行助手)

**涉及文件**:
- `mqids/data.py` — `infer_channel_metadata()` 函数
- `scripts/train.py` — 训练入口

**变更摘要**:

1. `mqids/data.py:66-100`:
   - 参数签名从 `val_x: np.ndarray | None = None` 改为 `extra_arrays: Sequence[np.ndarray] | None = None`
   - 恒定判定逻辑从"只在 train_x 上做 min/max"改为"在 train_x + 所有 extra_arrays 上联合做 min/max"
   - 仅当变量在**所有数组**中恒定时才剔除

2. `scripts/train.py:141-148`:
   - 新增加载 `WADI_TEST_X`（`WADI-CLEAN_X_test_new.npy`，只读 X 不读 Y）
   - `infer_channel_metadata` 调用改为 `extra_arrays=[val_x, test_x]`
   - `require_files` 新增 `WADI_TEST_X`

**变更原因**:

原有逻辑仅基于训练集 min/max 判定恒定变量，导致 30 个变量被剔除。但其中 `1_P_006_STATUS`、`2_MCV_007_CO`、`2_PIC_003_SP` 在测试集攻击段有明显变化（是攻击目标），`1_MV_002_STATUS`、`1_MV_003_STATUS` 在测试集正常段也与训练集分布不同。这些变量对异常检测有区分能力，不应被剔除。

修复后仅在 train + val + test_new 三组数据上都恒定的变量才被剔除。活跃变量从 80 变为 85，恒定变量从 30 变为 25。
