# WADI 离散变量文本化（DTT）开发集消融审计

## 实验性质

本批实验研究 `discrete_to_text`：54 个连续变量仍由 MOIRAI 编码为连续 Token，31 个二值/低基数变量改写为中文状态文本并插入 Qwen Prompt。实验均使用单攻击事件 support→query 开发协议、stride=8、6364 个端点和 85 个异常端点。

这些结果只能作为 **transductive 开发诊断**，不能作为严格隔离的正式测试证据。原因是当前训练入口读取 `WADI_TEST_X`，并将 train、validation、test X 联合用于通道恒定性判断，使活跃变量从训练集协议的 80 个变成 85 个。虽然没有读取测试标签，但模型配置依赖了测试分布；此外 `protocol.json` 仍错误记录 `test_arrays_opened=false`。

所有正式 DTT 运行目录均创建于 2026-07-21 09:19 之后，晚于批次文本 bug 修复时间 09:18，因此使用的是逐样本离散文本和 padding mask 修正版。

## DTT ON/OFF 三种子对照

固定 `L=64`、显式第12层、DirectProjector、分类头单损失和 process Prompt：

| 配置 | Seed PR-AUC | PR-AUC | F1 | FPR | 峰值显存 | 记录累计耗时 |
|---|---|---:|---:|---:|---:|---:|
| DTT OFF，85 个数值 Token | 0.2822/0.1581/0.1666 | **0.2023±0.0693** | 0.2785±0.0278 | 0.0451±0.0390 | 2450 MB | 624±3 s |
| DTT ON，54 Token + 31 变量文本 | 0.1676/0.1826/0.1784 | 0.1762±0.0078 | **0.2948±0.0054** | 0.0450±0.0249 | 4524 MB | 1807±14 s |

DTT ON − OFF 的逐 seed PR-AUC 差值为 `-0.1146/+0.0245/+0.0118`，均值约 `-0.0261±0.0769`。F1 均值提高约 0.0162，但 PR-AUC 没有稳定提升，且 DTT 显存约增至 1.85 倍、记录耗时约增至 2.89 倍。

因此不能声称离散变量文本化优于数值 Token；它更稳定的 F1 来自同一开发集选阈值，不能抵消排序能力和资源成本问题。

## 窗口与层

DTT ON、Direct、seed 2026 的窗口筛选：

| L | PR-AUC | F1 | FPR |
|---:|---:|---:|---:|
| 8 | 0.1342 | 0.2564 | 0.0460 |
| 16 | **0.1936** | 0.2921 | **0.0293** |
| 32 | 0.1686 | 0.2980 | 0.0556 |
| 64 | 0.1676 | **0.3009** | 0.0602 |
| 128 | 0.1307 | 0.2605 | 0.0588 |

L=16 补跑三种子后 PR-AUC 为 `0.2148±0.0227`、F1 为 `0.2841±0.0319`；L=64 为 `0.1762±0.0078/0.2948±0.0054`。DTT 模式下 L=16 的排序更好，但 F1 略低，说明最佳窗口会随输入模态改变。

第12层三种子 PR-AUC/F1 为 `0.1762±0.0078/0.2948±0.0054`，第10层为 `0.1326±0.0164/0.2203±0.0235`。第12层仍明显领先，层选择结论在 DTT 诊断中保持一致。

## Projector 与损失

固定 DTT ON、L=64、第12层、分类头单损失：

| Projector | PR-AUC | F1 | FPR |
|---|---:|---:|---:|
| Linear | 0.1550±0.0102 | 0.2597±0.0367 | 0.0372±0.0273 |
| Direct | 0.1762±0.0078 | 0.2948±0.0054 | 0.0450±0.0249 |
| Reprogramming | **0.2184±0.0358** | **0.3185±0.0219** | 0.0409±0.0134 |

在 DTT 输入下，Reprogramming 优于 Direct；这与早期非 DTT、L=32 单 seed 结果不同，说明映射层效果依赖输入模态。但当前仍是同一验证攻击事件，不能直接升级为最终主方法。

等权词表辅助 `classifier=1, vocab=1` 的主分类 PR-AUC 为 `0.2522±0.0947`，F1 为 `0.3219±0.0506`；三个 seed 的 PR-AUC 均高于 DTT Direct 基线，但 seed 2028 的 0.3615 造成很大波动。弱词表辅助和纯 verbalizer 的计划三种子对照 **尚未有效完成**：

- `dtt_abl_loss_vocab0.1_seed2028` 实际是 L=32、未显式指定层，不能与 seed 2026/2027 的 L=64、第12层合并。
- `dtt_abl_loss_verbalizer_seed2027/2028` 实际是 L=32、未显式指定层；只有 seed 2026 满足 L=64、第12层，其词表 PR-AUC 为 0.1734。

将 L=16、Reprogramming、`vocab=1` 组合后，三种子 PR-AUC 只有 `0.1508±0.0834`，低于各单项消融。组件收益不具可加性，大量基于同一开发事件的组合搜索已经表现出明显过拟合。

## 离散文本语义 bug

当前 `switch_templates.py` 在每个窗口内用排序后的局部唯一值重新编号状态。该实现会造成语义错误：

- 二值变量若整个窗口持续为原始值 1，窗口内唯一集合只有 `{1}`，它仍被编号为状态 0 并输出“关闭”；持续 0 也输出“关闭”。
- 低基数变量的“状态N”编号随窗口内出现过哪些值变化，同一个原始状态在不同窗口可能被写成不同状态编号。

因此现有 DTT 文本不是稳定、可信的工控状态描述。状态词表必须由训练数据或预定义设备语义一次性建立，不能按单窗重映射。该 bug 与测试 X 通道泄漏共同使当前 DTT 数值不适合进入正式测试。

## 当前结论与下一步

1. Prompt 反事实已经表明正确工控固定 Prompt 没有稳定语义优势；DTT 也未在当前实现下稳定优于数值 Token。
2. 暂停继续扩大 DTT 超参数网格。先恢复严格隔离：训练/开发入口不得读取测试 X/Y，通道集合必须在测试前锁定，且协议记录必须反映真实打开的数据。
3. 修复离散状态全局映射后，只需重跑最小的四项机制对照：全部数值 Token、仅连续 Token、连续 Token + 离散文本、全部数值 Token + 离散文本。这样才能区分“删除离散数值信息”和“新增文本信息”的贡献。
4. 修复后先做单 seed；只有机制对照显示稳定方向，再补三种子。当前不应执行 stride=1 锁定或正式 WADI 测试。
