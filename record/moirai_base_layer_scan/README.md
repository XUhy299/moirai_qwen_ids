# WADI MOIRAI Base 全层 Token 筛选与第12层多种子复验

## 实验性质与问题

本实验回答两个问题：

1. 在当前 `L=patch_size=64`、一变量一 Token、DirectProjector、冻结 Qwen3-0.6B
   架构下，MOIRAI Base 的第几层 Encoder 输出更适合作为时序 Token？
2. 胜出层相对“相同 MOIRAI Token + 无 Qwen 小 Transformer”是否存在稳定开发集增益？

本实验仍是 WADI 验证集单攻击事件 support→query 诊断，没有打开正式测试数组，不能表述为
独立攻击或测试集结论。实验日期：2026-07-19。

## 中间层实现与工程验证

- MOIRAI Base 包含 12 层 Transformer Encoder，实验层号使用 1-based 编号 `1..12`。
- 每个长度为 64 的变量 Patch 先经 MOIRAI `in_proj` 映射到 768 维，再执行到指定 Encoder 层。
- 对选中层统一施加 Encoder 最终 RMSNorm，然后选择 `prediction_mask == false` 的历史 Token。
- 每个有效变量恰好输出一个 Token，形状为 `[B,80,768]`；DirectProjector 映射到
  `[B,80,1024]` 后送入冻结 Qwen。
- 显式第12层与原始完整 `module.encoder(...)` 路径已在真实 WADI L=64 窗口上验证：输出形状
  `[1,80,768]`，最大绝对误差 `0.000e+00`，因此复用窗口网格中的原最终层运行作为第12层
  seed 2026 结果，不重复计算。
- 真实第6层 MOIRAI→Direct→Qwen 训练 smoke 已通过。

## 数据与固定配置

- 正常训练：510 个窗口；异常 support：170 个窗口；总训练窗口 680 个。
- 开发：6364 个 stride=8 端点，其中异常 85 个。
- 统一 `common_min_endpoint=127`，标签为 `Y[window_end]`。
- 层扫描固定：L=64、DirectProjector、分类头单损失、seed 2026、5 epoch。
- 12 层的正常/异常/开发端点 SHA256 分别只有一个唯一值，配置间样本完全一致。
- 所有层按开发 PR-AUC 选择 epoch，其他指标均取同一 epoch。
- 所有 `protocol.json` 均为 `test_arrays_opened=false`。

## 资源与校准问题

- Qwen 路径可训练参数 2,846,722，无 Qwen 对照 1,275,138；冻结参数不计入这两个数字。
- Qwen 峰值显存约 2401 MB，无 Qwen约 496 MB，前者约为 4.8 倍。
- Qwen 五轮累计训练+开发耗时约为无 Qwen 的 3.3 倍。
- 第12层 Qwen 三个 seed 的最佳阈值分别为 `0.9955/0.9358/0.1520`，差异极大，说明分类概率
  校准不稳定。当前主要证据是 PR-AUC 排序，不能直接迁移任一阈值到正式测试。

## 结论边界

1. 第12层是当前 L=64、Direct、seed 2026 全层网格的最佳候选，并在三个 seed 上得到可重复
   的开发集信号；但只有第12层做了多种子，尚未证明它的均值一定高于第10层均值。
2. Qwen 相对当前无 Qwen基线三种子均提高 PR-AUC，但实验仍只有一个验证攻击事件，不能说明
   跨攻击类型泛化，也不能声称正式测试优于 AE。
3. support/query 来自同一攻击事件；当前阈值与 epoch 都依赖开发集，存在验证过拟合风险。
4. 正式 WADI 测试数组仍未打开，本实验不能与 AE 测试 F1=0.6334 做直接性能排名。

## 下一步计划

1. 用 seed 2027/2028 复验第二名第10层，确认第12层的层选择优势是否跨 seed 稳定。
2. 固定最终候选层和 L=64，实施正确工控 Prompt、通用 Prompt、删除 Prompt、错误/打乱 Prompt
   的反事实消融；先单种子筛选，再对关键对照多种子复验。
3. 研究验证集阈值校准与稳定性，但不得使用测试标签；优先报告 PR-AUC、FPR和阈值波动。
4. 在选定配置上运行 stride=1 验证，生成点级分数、事件级指标和检测延迟，并锁定 checkpoint、
   阈值、标签规则与后处理。
5. 新增只读锁定配置的正式测试入口，确认其不含测试调参逻辑后，一次性运行全量 WADI 测试。
6. 只有锁定测试证实相对 AE、传统基线或无 Qwen基线的优势后，才启动 Student 蒸馏与端侧导出。

## 2026-07-20：第12层显式复现与第10层三种子确认

### 可比性审计

- `scripts/check_moirai_layer_equivalence.py` 已在真实 WADI、L=64 单窗上验证：旧的完整 `module.encoder(...)` 最终输出与当前显式指定第12层的输出均为 `[1,80,768]`，最大绝对误差为 `0.000e+00`。
- 为排除第12层曾单独使用旧调用路径造成的疑问，采用当前显式第12层代码重跑 seed 2026。其 PR-AUC/F1 为 `0.2059/0.3559`，而旧路径为 `0.2139/0.3626`，差异仅为 `-0.0080/-0.0067`。两个运行的数据协议及端点哈希相同；这说明旧第12层结果与当前逐层实现方法学可比，但三种子层选择统一使用显式复现的值。

### 锁定结论与下一步

L=64、显式 MOIRAI Base 第12层现被锁定为 Prompt 反事实消融的候选层。此选择只由同一验证攻击事件的开发集信号支持：三种子尚不能证明跨攻击、跨数据集或正式测试优势，也不能解释为 Prompt 或 ICS 知识已经有效。

下一项实验只改变 Prompt：正确过程知识、通用任务、最简/删除知识、错误/打乱知识在相同数据、seed、训练步数和选模规则下进行单种子筛选；仅对关键对照再做多种子复验。随后才进行 stride=1 阈值锁定和一次性正式测试。
