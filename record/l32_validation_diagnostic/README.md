# WADI MOIRAI-Qwen L=32 单攻击事件验证诊断

## 实验性质

本实验用于验证“冻结 MOIRAI 变量级 Token + 映射层 + 冻结 Qwen3-0.6B”路线是否值得继续研究。
它是**验证集单攻击事件 support→query 诊断**，不是正式测试集结果，也不是无监督异常检测结果。
实验过程中没有打开 WADI 的 `X_test_new/Y_test_new`。

实验日期：2026-07-19。

## 数据与切分

- 正常源：`WADI-CLEAN_X_train.npy`，每个 seed 从正常训练端点中随机抽取 510 个窗口。
- 异常源：WADI 验证集唯一攻击段 `[5139, 6625)`。
- support 异常：攻击前半段按 stride=4 取 170 个窗口端点。
- 隔离区：`[5818, 5946)`，宽度 128，不允许 support/query 窗口跨越。
- query/开发集：攻击后半段及相邻正常区按 stride=8 评估，共 6376 个端点，其中异常 85 个（1.33%）。
- 标签规则：窗口端点标签 `Y[s + L - 1]`。
- 窗口：`L=patch_size=32`，每个有效变量对应一个 MOIRAI context Token。
- WADI 110 个原始变量中，按完整正常训练集删除 30 个恒定变量，保留 80 个有效变量：54 个连续、16 个二值、10 个低基数变量。

support 与 query 仍来自同一个攻击事件。128 点隔离只能避免重叠窗口泄漏，不能提供独立攻击事件泛化证据。

## 模型与训练

- MOIRAI Base：冻结，输出 `[B, 80, 768]`。
- Qwen3-0.6B：冻结，隐藏维度 1024。
- Prompt 顺序：固定工控任务前缀 → 80 个连续时序 Token → 判定后缀。
- 主分类输出：最后判定位置隐藏状态经轻量二分类头输出正常/异常概率。
- 词表输出：只在 Qwen 的单 Token 标签“正常”“异常”之间归一化。
- 优化：AdamW，学习率 `3e-4`，weight decay `1e-4`，训练 batch=4，评估 batch=16，最多 5 epoch。
- 模型选择：开发集 PR-AUC 最高 epoch；纯 verbalizer 消融按 `vocab_pr_auc` 选择。
- 报告 F1：使用所选 epoch 在同一开发集上选择的最佳阈值。不能把不同 epoch 的最高 F1 与最高 PR-AUC 任意组合。
- 环境：Windows，Python 3.11.9，PyTorch 2.4.1+cu121，Transformers 4.51.3，RTX 4060 Laptop 8GB。

## 发现

1. 真实整栈工程闭环成立：MOIRAI Base 对一个 `[1,32,80]` 窗口输出 `[1,80,768]`，映射后为 `[1,80,1024]`；Qwen/MOIRAI 参数无梯度，映射层能通过冻结 Qwen 获得有限非零梯度。
2. DirectProjector 在三个 seed 上均提高开发 PR-AUC，说明“保留变量 Token 后通过冻结 Qwen 非线性变换”值得继续验证；但它同时提高平均 FPR，并带来约 4.8 倍峰值显存和约 3.1 倍训练/开发耗时。
3. 仅增加无 Qwen 模型容量没有改善结果：hidden=384 参数匹配基线在 seed 2026 明显恶化，不能用参数量简单解释 Direct 的提升。
4. Reprogramming 文本原型交叉注意力没有超过 Direct；当前不能把它作为核心创新点。
5. `0.1 × verbalizer CE` 确实能把词表 PR 从接近随机提高到部分 epoch 的 0.16--0.19，但没有提高主分类头的最佳 PR。纯 verbalizer 的 seed 2026 PR 为 0.1472，表现有信号但训练波动大，适合作为消融而非当前主输出。
6. 所有模型训练损失下降时开发 PR 经常回落，说明单事件少样本监督过拟合明显，必须使用开发 PR 早停和多种子报告。

## 局限性与下一步

- 本实验只使用 L=32；尚未比较用户限定的 `8/16/64/128`。
- support/query 属于同一攻击事件，不能证明跨攻击类型泛化。
- 开发集使用 stride=8；正式锁定前需在 stride=1 验证上重算分数和阈值。
- 未进行正确工控 Prompt、通用 Prompt、错误/打乱 Prompt 的反事实消融，尚不能声称工控知识 Prompt 有贡献。
- 尚未运行锁定 WADI 测试集，也未报告事件级指标和检测延迟。
- 当前结果不能与既有 AE 的正式测试 F1 直接比较，更不能声称超过基线。
- 下一步优先用 DirectProjector 在 L=`8/16/64/128` 做单种子开发筛选；再对最佳窗口进行三种子复验和 Prompt 反事实消融。只有协议锁定后才运行一次 WADI 测试。
