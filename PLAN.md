# MOIRAI-Qwen IDS 后续实验计划

更新时间：2026-07-23

## 使用规则

- 未经用户在当前对话中明确许可，只运行编译、静态检查、prepare-only 和 smoke；完整训练还必须传入 `--full-run-authorized`，stride=1 与正式测试需单独获准。
- `[ ]` 表示尚未完成，`[x]` 表示已经完成并经过产物与配置复核。
- 仅仅启动运行、完成 smoke 或得到部分 seed 结果，不能勾选完整实验。
- 勾选实验前，必须确认输出目录包含配置、协议、逐 epoch 指标和 `results.md`。
- 较大实验完成后，还必须更新对应的 `record/<实验名>/README.md`；需要迁移的约束同步写入 `AGENTS.md`。
- 若实验因泄漏、配置漂移或实现 bug 失效，保留任务并注明原因，不得以“运行结束”代替“有效完成”。

- [x] 将 DTT 输入改为逐变量交错布局：变量名文本后紧跟对应连续 soft Token，离散变量名后紧跟当前状态文本。

## 阶段 C：最小 DTT 机制对照

固定同一训练期通道集合、support/query、seed 2026、相同选模规则，先只比较：

- [x] 实现 D 的独立输入开关：全部变量保留数值 Token，并为离散变量额外插入末端状态文本；已完成代码与合成 smoke，尚无效果结果。
- [x] 实现 synthetic 每 epoch 确定性按算子分层轮换，协议保存逐 epoch 索引哈希、算子分布和跨 epoch 唯一窗口数。
- [ ] A：全部活跃变量作为数值 MOIRAI Token，不添加离散文本。
- [ ] B：仅连续变量作为数值 MOIRAI Token，不添加离散文本。
- [ ] C：连续变量数值 Token + 修复后的离散变量文本。
- [ ] D：全部变量数值 Token + 离散变量文本，用于检验文本是否提供增量信息。
- [ ] 汇总 A/B/C/D 的 PR-AUC、F1、FPR、显存、耗时和 Prompt token 数。
- [ ] 判断文本收益来自“新增文本信息”还是“删除离散数值 Token”。
- [ ] 将结果写入 `record/dtt_mechanism_controls/README.md`。

决策门：

- [ ] 若 C/D 相对相应无文本对照没有稳定排序收益，停止 DTT 路线，不再扩大窗口、Projector 或损失网格。
